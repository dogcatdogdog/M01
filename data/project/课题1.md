## 1. 引言

### 1.1 编写目的

本文档定义「无人机续航预估与自适应校准系统」的功能与非功能需求，作为设计、开发、测试与验收的依据。

**核心要解决的问题**：不同型号（甚至同型号不同个体）的无人机，其重量、动力、电池、老化程度各不相同；如何在不为每个型号手写一套计算逻辑的前提下，对任意机型做出**足够准确的续航/航程预估**，并且**随真实飞行数据积累自动变准**，同时始终**预留足够的返航电量以保证安全**。

### 1.2 项目背景

任务规划时需要回答三个问题：

1. **能不能飞**：给定航点、爬升高度、盘旋时长、气象条件，电量是否够？
2. **能飞多远 / 能盘多久**：给定电量上限，最大航程 / 作业时长是多少？
3. **能不能安全回来**：在任意航点，剩余电量是否足以返航 + 降落？

现有规划多依赖固定经验值或厂家标称值，误差大、保守过度，导致有效航程被浪费；直接套用物理公式又因缺乏实测标定而不准；且常把"返航电量"当作一个固定百分比，忽略了不同航点、不同气象下返航能耗的差异。本系统通过「**通用物理模型 + 每架飞机性能档案 + 实测校准 + 返航电量动态预留**」系统性解决上述问题。

### 1.3 系统范围

**纳入范围（In Scope）**

- 飞机性能档案的建模与管理
- 基于物理模型与规格书的冷启动预估
- 分阶段（爬升 / 盘旋 / 巡航 / 降落）能耗计算与气象修正
- **返航电量（RTH Reserve）的动态预留与触发电量阈值计算**
- 飞行日志采集、切片与特征提取
- 基于实测数据的参数辨识与在线自适应更新
- 电池健康度（SOH）跟踪
- 基于不确定性的动态安全余量
- 对外服务接口

**不纳入范围（Out of Scope）**

- 航线本身的路径规划算法（避障、空域禁飞）
- 飞行控制系统（飞控）及其在飞行中的自动返航执行
- 气象预报数据的生产（本系统只消费）
- 多机协同调度

> 说明：本系统在规划期计算「返航电量预留」与「RTH 触发电量阈值」并输出；飞行中实际触发返航的动作由飞控/监控模块负责，不在本系统范围。

### 1.4 术语与缩略语

| 术语                            | 含义                                                       |
| ------------------------------- | ---------------------------------------------------------- |
| Endurance / 续航                | 单次起飞可停留空中的总时长                                 |
| Range / 航程                    | 单次起飞可飞行的最大水平距离                               |
| 性能档案 Profile                | 单架飞机（按序列号）的一组参数集合                         |
| DoD                             | 放电深度，可用电量占标称容量的比例                         |
| SOH                             | 电池健康度，当前实际容量相对全新容量的比例                 |
| 空速 / 地速                     | 相对空气 / 相对地面的速度                                  |
| 冷启动                          | 飞机尚无实测数据时的初始预估                               |
| 校准 / 参数辨识                 | 用实测数据反推模型系数的过程                               |
| 残差                            | 模型预测值与实测值的相对偏差                               |
| **RTH / 返航**                  | Return To Home，返回起飞点（或指定降落点）                 |
| **返航电量预留（RTH Reserve）** | 为保证从任意航点安全返航+降落而预留的电量                  |
| **RTH 触发电量阈值**            | 剩余电量降至该值时应触发返航（本系统算出供监控使用）       |
| **能量最不利点**                | 全航线中返航能耗最高的航点（受风向影响，未必是几何最远点） |
| **不确定性余量**                | 应对模型误差的统计性余量（随校准精度变化）                 |

## 2. 总体描述

### 2.1 系统定位

本系统是一个**独立的计算与学习引擎**：对上游接收航线/任务/气象，输出续航、航程、返航电量预留、RTH 触发阈值与可行性判断；在下游持续接收飞行日志，回写更新各飞机的性能档案，使预估随使用越来越准。系统可独立部署为服务，也可作为库嵌入宿主系统。

```
            ┌─────────────────────────────────────────────┐
航线/任务 ─▶│         续航预估与自适应校准系统            │──▶ 续航/航程 + 返航电量 + RTH 阈值 + 可行性
气象数据 ──▶│  (预估引擎 + 校准引擎 + 性能档案库 + 日志)  │
飞行日志 ──▶│                                              │──▶ 回写更新性能档案
            └─────────────────────────────────────────────┘
```

### 2.2 角色与使用场景

| 角色                       | 场景                                                         |
| -------------------------- | ------------------------------------------------------------ |
| 任务规划员                 | 制定航线前，调用预估判断可行性、盘旋时长上限、返航电量是否充足、安全余量 |
| 运维工程师                 | 新飞机入库时录入规格书生成冷启动档案；查看/调校性能参数      |
| 飞行监控模块（外部消费方） | 读取本系统输出的 RTH 触发电量阈值，用于飞行中的返航告警      |
| 系统（自动）               | 每次飞行结束后自动采集日志、校准系数、更新 SOH               |

### 2.3 运行环境与约束

- 部署形态：**独立可部署服务 / 可嵌入库**，无强宿主耦合
- 机型支持：多旋翼、固定翼（两类物理模型，可插拔）
- 约束 C-1：预估必须**保守优先**，宁可低估续航、不可高估
- 约束 C-2：每次预估需在 500ms 内返回（规划阶段可能批量试算）
- 约束 C-3：数据不足时必须放大安全余量与返航预留，不得给出虚假的高精度
- 约束 C-4：**返航电量预留为硬约束**，任何情况下不得为零或低于硬下限

### 2.4 核心设计思想（四层模型）

预估精度 = **通用物理模型** + **每架飞机性能档案（随飞行更新）** + **不确定性→动态余量** + **返航电量动态预留**。

- 型号差异不靠「换一套公式」，而靠「同一公式代入不同参数」；
- 数据少时余量与返航预留大（保守），数据多、模型准时收紧不确定性余量（释放航程），但返航预留仍由实际返航能耗决定；
- 把飞行数据转化为可直接变现的航程收益，同时始终保证「能安全回来」。

## 3. 功能需求

> 优先级定义：**P0**=必须有，上线阻断；**P1**=重要，首发应含；**P2**=增强，可迭代。

### FR-1 飞机性能档案管理

| 项     | 内容                                                         |
| ------ | ------------------------------------------------------------ |
| 编号   | FR-1                                                         |
| 优先级 | P0                                                           |
| 描述   | 以「机身序列号」为粒度，为每架飞机维护一份可演进的性能档案，作为所有预估与校准的输入。 |

**参数分组**

1. **结构参数（出厂固定）**：序列号、型号、机型类别（多旋翼/固定翼）、最大起飞重量 MTOW、电池标称容量（Wh）、旋翼总盘面积 / 机翼面积、最佳巡航空速、最大爬升率、返航/降落点默认配置。
2. **性能系数（可标定）**：悬停效率系数 `k_hover`、爬升功率倍数 `k_climb`、巡航功率倍数 `k_cruise`。
3. **电池状态**：SOH、充放电循环次数。
4. **安全策略参数**：DoD、返航安全系数 `k_return`、返航电量硬下限（%）、不确定性余量硬下限（%）。
5. **不确定性元数据**：累计校准样本数 `samples`、预测残差标准差 `residual_std`、最后更新时间。

**业务规则**

- 同一序列号档案唯一，结构参数变更需留版本记录；
- 性能系数支持手动覆盖（运维调校），手动值需打标记，不被自动校准覆盖；
- 档案变更需可追溯（审计日志）。

**验收标准（AC）**

- AC1：可通过序列号查询/创建/更新档案，字段完整；
- AC2：结构参数与性能系数分区存储，手动覆盖标记生效；
- AC3：每次变更写入审计记录，含操作人、时间、前后值。

### FR-2 冷启动预估

| 项     | 内容                                                         |
| ------ | ------------------------------------------------------------ |
| 编号   | FR-2                                                         |
| 优先级 | P0                                                           |
| 描述   | 新飞机无实测数据时，根据规格书 + 物理公式生成初始性能档案，保证可用但偏保守。 |

**初始值生成策略（精度从高到低）**

1. **规格书反推**：由厂家标称「最大续航 / 最大航程 / 悬停时间」反推平均功率与各类系数；
2. **物理公式**：多旋翼动量理论 `P_hover=(mg)^1.5/√(2ρA)`；固定翼 `P=½ρV³SCd/η`；
3. **经验法则兜底**：按机型级别给定单位重量悬停功率（消费级 ~200 W/kg，工业级 ~150 W/kg，重载 ~120 W/kg）。

**业务规则**

- 冷启动档案 `samples=0`，`residual_std` 初始取较大值（默认 0.20），强制放大不确定性余量与返航预留；
- 返航安全系数 `k_return` 在冷启动阶段取偏保守默认值（如 1.3）。

**验收标准（AC）**

- AC1：仅录入规格书即可生成完整档案，无空字段；
- AC2：冷启动档案的预估续航不低于实际续航（保守性，统计意义上）；
- AC3：缺关键字段时给出明确告警与降级说明，而非静默用零值。

### FR-3 续航与航程预估计算

| 项     | 内容                                                         |
| ------ | ------------------------------------------------------------ |
| 编号   | FR-3                                                         |
| 优先级 | P0                                                           |
| 描述   | 给定航线（航点序列、各段高度、飞行模式、盘旋时长）、性能档案、气象数据，输出总能耗、续航时间、航程、返航电量预留、可行性判断。 |

**输入**：航点列表 `[{x, y, alt, mode, hover_s}]`，`mode ∈ {start, climb, hover, cruise, land}`；飞机档案；气象数据。

**电量预算模型（核心判据）**

```
E_usable  = 电池Wh × DoD × SOH                      # 总可用电量
E_task    = Σ 各阶段(爬升/盘旋/巡航/降落) 能耗      # 完成任务所需
E_return  = 返航电量预留 (见 FR-11)                  # 保证能回来
E_margin  = 不确定性余量 (见 FR-9)                   # 应对模型误差

可行性判据:  E_task + E_return + E_margin  ≤  E_usable
```

**分阶段能耗（E_task 构成）**

- 可用电量 `E_usable = 电池Wh × DoD × SOH`；
- **爬升**：`E_climb = m·g·h/η_climb + P_hover·k_climb·t_climb`；
- **盘旋/悬停**：`E_hover = P_hover·k_hover·(1+盘旋修正)·t`；绕圈飞行比纯悬停高 10~20%；
- **巡航**：地速 = 空速 + 航向风分量；`E_cruise = P_hover·k_cruise·t`，`t=距离/地速`；
- **降落**：保守按爬升能耗的 30~50% 计，不计回收；
- 悬停功率 `P_hover = k_hover·(m·g)^1.5/√(2·ρ·A)`，`ρ` 由高度与温度计算。

**输出**：总任务能耗（Wh）、预计续航（min）、预计航程（km）、返航电量预留 E_return（Wh）、RTH 触发电量阈值（%）、可行性布尔、各阶段能耗明细、瓶颈段。

**业务规则**

- 逐航段计算并叠加气象修正（见 FR-5）；
- 不可行时返回原因：是 E_task 超支、还是 E_return/E_margin 不足，并给出建议（削减盘旋时长、缩短航程、提高起飞电量）。

**验收标准（AC）**

- AC1：给定标准航线，输出各阶段能耗明细且合计守恒；
- AC2：往返航线逆/顺风分别计算，不复用单程结果；
- AC3：可行性判据同时考虑任务能耗、返航预留、不确定性余量三项；
- AC4：电量不足时明确指出瓶颈维度（任务/返航/余量）与可操作建议。

### FR-4 气象数据接入与修正

| 项     | 内容                                                         |
| ------ | ------------------------------------------------------------ |
| 编号   | FR-4                                                         |
| 优先级 | P1                                                           |
| 描述   | 接入航线高度层的风、温度数据，修正地速、空气密度、电池可用容量；返航能耗按返航气象单独计算。 |

**修正项**

- **风**：按航段方向分解风分量，修正地速与空速；往返分段计算；**返航段按返航方向的风分量计算**（去程顺风 ≠ 返航也顺风）；
- **温度**：低温下调电池可用容量（每降 10°C 约 −3~5%），高温触发过热降功率告警；
- **空气密度 ρ**：`ρ=ρ₀·(P/P₀)·(T₀/T)`，影响升力与悬停功率；高海拔/高温场景显著；
- **降水/能见度**：仅作安全告警，不计入能耗。

**业务规则**

- 取**飞行高度层**的风温，禁用地面值直接代入；
- 气象数据缺失时降级为无风预估并增大气象余量与返航预留。

**验收标准（AC）**

- AC1：同一航线，逆风预估能耗高于顺风；
- AC2：高原场景（如海拔 3000m）悬停功率较海平面明显上升；
- AC3：气象缺失时不报错，而是降级 + 标注不确定性上升；
- AC4：返航能耗使用返航方向的风分量，与去程不对称。

### FR-5 飞行日志采集与切片

| 项     | 内容                                                         |
| ------ | ------------------------------------------------------------ |
| 编号   | FR-5                                                         |
| 优先级 | P0（校准的燃料，最关键前置）                                 |
| 描述   | 采集飞行过程的时序数据，按飞行模式切片，提取用于校准的特征样本。 |

**采集字段（建议最小集）**

时间戳、GPS（经纬高）、空速、地速、爬升率、电池电压/电流（→瞬时功率）、电池温度、环境温度、载重、飞行模式标记、各航点到达时刻。

**采样与切片**

- 采样频率：≥1Hz（电流/电压建议 ≥10Hz）；
- 切片：按 `mode` 标记将整段飞行切为 climb/hover/cruise/land 子段；
- 每个子段聚合为一条样本：`(质量, 空速, 爬升率, ρ, 温度, 时长) → 该段平均功率`；
- 质量过滤：剔除起降瞬态、阵风尖峰、传感器异常段。

**业务规则**

- 一条日志可产出数十至上百条样本；
- 异常段（电流突变、GPS 跳变）必须剔除，不得污染校准。

**验收标准（AC）**

- AC1：能解析至少一种日志格式并产出结构化样本；
- AC2：切片边界准确，各段功率与总耗电守恒（误差 < 阈值）；
- AC3：异常段被识别并标记剔除原因。

### FR-6 实测校准（参数辨识）

| 项     | 内容                                                         |
| ------ | ------------------------------------------------------------ |
| 编号   | FR-6                                                         |
| 优先级 | P1                                                           |
| 描述   | 用切片样本反推性能系数，并对每个系数估计其残差（不确定性）。 |

**方法**

- 悬停系数：理论关系对 `k_hover` 线性 → 最小二乘 `lstsq`；
- 爬升/巡航系数：多元线性或非线性（如巡航功率随空速三次方）→ `curve_fit`；
- 输出：各系数的拟合值 + 残差标准差 + 样本数。

**业务规则**

- 样本数不足（如 < 30）时不强制更新，仅记录；
- 拟合结果异常（如系数偏离物理合理区间）时告警并拒绝写入，保留旧值；
- 手动覆盖标记的系数不参与自动校准。

**验收标准（AC）**

- AC1：给定带噪样本，拟合系数收敛到真值附近（误差在可接受区间）；
- AC2：异常拟合（负功率系数等）被拒绝并告警；
- AC3：每次校准产出包含「系数 + 残差 + 样本数 + 是否采纳」的记录。

### FR-7 在线自适应更新

| 项     | 内容                                                         |
| ------ | ------------------------------------------------------------ |
| 编号   | FR-7                                                         |
| 优先级 | P1                                                           |
| 描述   | 每次飞行校准后，用指数加权将新系数并入档案，使模型随飞行演进、且对新数据更敏感。 |

**方法**

- 指数加权移动更新：`新值 = (1-α)·旧值 + α·本次校准值`，默认 α=0.3；
- 同步更新 `samples` 累加与 `residual_std` 的滚动估计；
- 更新写审计日志。

**业务规则**

- α 可配置，过大易震荡，过小学习慢；
- 可选闭环：仅当本次校准残差优于历史时才更新（稳健更新）。

**验收标准（AC）**

- AC1：连续多次飞行后，档案系数单调收敛、残差下降；
- AC2：单次异常校准不会使系数剧跳（被 α 平滑或被 FR-6 拒绝）；
- AC3：更新全链路可追溯。

### FR-8 电池老化跟踪

| 项     | 内容                                                         |
| ------ | ------------------------------------------------------------ |
| 编号   | FR-8                                                         |
| 优先级 | P2                                                           |
| 描述   | 跟踪电池随循环次数的容量衰减，使预估与返航预留自动反映老化。 |

**方法**

- 每次飞行后用「实际放出 Wh / 标称 Wh」更新 SOH；
- 维护 SOH 随循环次数的衰减曲线（锂电池典型每 100 循环 −3~5%）；
- SOH 直接进入可用电量公式 `E_usable = 电池Wh × DoD × SOH`。

**业务规则**

- SOH 跌破阈值（如 0.8）触发「建议更换电池」告警；
- 更换电池后可重置 SOH 与循环计数。

**验收标准（AC）**

- AC1：模拟循环累积后 SOH 单调下降，且预估续航与返航预留同步下降；
- AC2：SOH 低于阈值时产生告警；
- AC3：更换电池后可重置统计。

### FR-9 不确定性动态余量

| 项     | 内容                                                         |
| ------ | ------------------------------------------------------------ |
| 编号   | FR-9                                                         |
| 优先级 | P0                                                           |
| 描述   | 应对模型误差的统计性余量（E_margin）不写死，而按「数据量 + 残差 + 气象波动」动态计算。 |

**公式（示意）**

```
E_margin = E_usable × max( 下限(15%), k·residual_std·√(1/samples) ) + 气象波动余量
```

- 新飞机（samples=0, residual_std=0.20）→ 余量大；
- 飞行充分后残差降到 5% → 余量收紧 → 同样电池飞更远。

**业务规则**

- 余量有硬下限，任何情况不得低于该下限；
- **E_margin 与返航电量 E_return 是两类不同预留、相互叠加**：E_margin 应对"能耗预测算错了"，E_return 应对"必须留够回家的电"；
- 余量计算过程对规划员透明可见（为何这么保守）。

**验收标准（AC）**

- AC1：同一飞机，样本量与残差改善后，相同航线的 E_margin 收紧、可用航程提升；
- AC2：任何情况下 E_margin 不低于硬下限；
- AC3：余量组成（数据项、气象项）可解释输出；
- AC4：E_margin 与 E_return 分别列示、叠加计算，不互相吸收。

### FR-11 返航电量预留与 RTH 触发阈值

| 项     | 内容                                                         |
| ------ | ------------------------------------------------------------ |
| 编号   | FR-11                                                        |
| 优先级 | P0                                                           |
| 描述   | 动态计算返航电量预留（E_return）与 RTH 触发电量阈值，保证从任意航点都能安全返航+降落，作为可行性判据的硬约束。 |

**为什么需要动态预留（而非固定百分比）**

- 返航能耗取决于「从哪个点返回」「返回时的气象（风）」「飞机当前性能与载重」；
- 逆风去程的航点，返程可能逆风更严重；几何最远的点不一定是返航能耗最高的点；
- 固定 30% 的写法在不同航线/气象下要么不足（危险）要么浪费（航程损失）。

**计算步骤**

1. **遍历航线所有航点**，对每个航点 i，按**返航方向气象**计算从该点返回降落点的能耗 `E_return_i`（含返航巡航 + 降落，使用 FR-4 的风分量修正）；
2. **识别能量最不利点** `i* = argmax(E_return_i)`，即返航能耗最大的航点；
3. **返航电量预留** `E_return = E_return_{i*} × k_return`（`k_return` 为返航安全系数，默认 ≥1.2，冷启动更保守）；
4. **保底约束**：`E_return ≥ 电池Wh × DoD × SOH × 返航电量硬下限(%)`（如 10%），取两者较大值；
5. **RTH 触发电量阈值**：将 E_return 换算为剩余电量百分比 = `E_return / (电池Wh × DoD × SOH) × 100%`，供飞行监控模块使用。

**输入**：航点列表、飞机档案（含 k_return、硬下限）、气象。

**输出**：返航电量预留 E_return（Wh）、各航点返航能耗曲线、能量最不利点位置、RTH 触发电量阈值（%）。

**业务规则**

- E_return 必须计入可行性判据（FR-3），为硬约束，不得为零或低于硬下限（约束 C-4）；
- 返航能耗一律按返航方向气象计算，不得复用去程结果；
- 气象缺失时，返航能耗按"保守假设"（如全程逆风）估算并标注；
- 当返航预留过大导致任务不可行时，提示缩短航程/降低盘旋/提高起飞电量，而非削减返航预留。

**验收标准（AC）**

- AC1：逆风去程场景下，能量最不利点的返航能耗高于无风情况，E_return 相应增大；
- AC2：E_return 始终 ≥ 硬下限，且为可行性判据的必要项；
- AC3：RTH 触发电量阈值 = E_return 对应百分比，可被监控模块直接消费；
- AC4：气象缺失时按保守假设计算并标注，不报错也不给出过于乐观的 E_return；
- AC5：当仅任务能耗够但 E_return 不足时，明确判定为"不可行（返航电量不足）"而非笼统失败。

## 4. 非功能需求

| 编号  | 类别        | 要求                                                         |
| ----- | ----------- | ------------------------------------------------------------ |
| NFR-1 | 性能        | 单次预估 ≤ 500ms；批量试算（如 50 条航线）≤ 3s               |
| NFR-2 | 可靠性      | 校准失败不得影响预估可用（降级用旧档案）；日志解析异常不得中断服务 |
| NFR-3 | 安全/保守性 | 任何预估不得系统性高估续航（统计意义上偏保守）；返航预留不得被绕过 |
| NFR-4 | 可维护性    | 物理模型与数据访问解耦；新增机型只需补结构参数与模型类别     |
| NFR-5 | 可追溯性    | 档案、余量、返航预留的每一次变更可审计、可解释               |
| NFR-6 | 可扩展性    | 物理模型（多旋翼/固定翼）可插拔；未来支持混合翼等新机型      |
| NFR-7 | 数据一致性  | 同一飞机同一时刻的预估结果可复现（确定性输入→确定性输出）    |
| NFR-8 | 独立性      | 系统不依赖特定上层平台，可独立部署或作为库嵌入               |

## 6. 业务流程

### 6.1 规划阶段（预估 + 返航校验）

```
规划员提交航线 + 气象
    │
    ▼
读取飞机性能档案
    │
    ├─▶ 分阶段任务能耗 E_task (+ 气象修正)
    ├─▶ 返航电量预留 E_return (遍历航点, 取能量最不利点)  ──▶ RTH 触发阈值
    └─▶ 不确定性余量 E_margin (按残差/样本/气象动态)
    │
    ▼
可行性判断: E_task + E_return + E_margin ≤ E_usable ?
    │
    ▼
返回: 续航/航程 + 返航电量预留 + RTH 阈值 + 可行性 + 瓶颈维度 + 建议
```

### 6.2 学习阶段（校准）

```
飞行结束 ──▶ 采集日志 ──▶ 切片 + 特征提取
    │
    ▼
参数辨识(拟合系数 + 残差) ──▶ 合理性校验
    │            └─异常─▶ 拒绝 + 告警, 保留旧值
    ▼ 正常
指数加权更新档案 ──▶ 更新 SOH / cycles ──▶ 写审计
    │
    ▼
下次规划: 模型更准 → E_margin 收紧; 但 E_return 仍由实际返航能耗决定
```
