## 1. 引言

### 1.1 编写目的

本文档定义「巡航任务覆盖范围与优化系统」的功能与非功能需求，作为设计、开发、测试与验收的依据。

**核心要解决的问题**：对于航拍/测绘/巡检这类「需要把一片区域拍完」的巡航任务，如何在**不同飞行高度**下做出最优规划——**最省电、路线最短、覆盖范围最广**，同时算清：飞多高、盘旋/巡航多久、拍多少张照片、耗多少电、占多少存储、是否需要实时无线传输，并在飞行中**实时监控电量损耗、及时触发返航**。

### 1.2 项目背景

巡航任务与点对点飞行不同，它追求的是「用有限电量把目标区域完整覆盖」。这里存在一组强耦合的权衡：

- **高度 ↑** → 单张照片覆盖面积 ↑ → 需要照片 ↓ → 航程 ↓ → 省电；但分辨率 ↓；
- **分辨率要求** → 限制了最高飞行高度；
- **拍照数量 ↑** → 存储压力 ↑ → 可能需实时传输 → 传输又耗电、耗时；
- **电量有限** → 必须在「拍完区域」与「能安全返航」之间分配。

现有做法往往凭经验定高度和重叠率，导致要么分辨率不达标、要么照片太多塞爆存储、要么拍到一半没电。本系统把这些维度**统一建模、联合优化**，并接入课题 1 的能耗/返航能力做电量预算与实时监控。

### 1.3 系统范围

**纳入范围（In Scope）**

- 相机/传感器参数建模
- 覆盖范围与地面分辨率（GSD）计算
- 航高优化（最省电 / 最短航程 / 最广覆盖）
- 航线与重叠率规划、拍照数量与拍摄节奏计算
- 电量预算（复用课题 1 模型）
- 存储容量预算
- 无线传输决策与代价估算
- 多目标方案优化与推荐
- 实时电量监控与返航决策（读取电量 → 比对返航所需 → 触发返航）

**不纳入范围（Out of Scope）**

- 续航/能耗的基础物理模型与校准（由课题 1 提供）
- 飞控、相机硬件控制、实际拍照/传输执行
- 气象预报数据生产（仅消费）
- 影像后处理（拼图、三维建模）

## 2. 总体描述

### 2.1 系统定位

本系统是面向「区域覆盖类巡航任务」的**规划与监控引擎**：上游输入目标区域、相机参数、分辨率要求、气象；输出最优航高、航线、拍照计划、电量/存储/传输预算与可行性；飞行中持续监控电量并决策返航。能耗与返航电量计算下沉到课题 1 引擎。

```
目标区域 AOI ─┐
相机参数 ─────┤
分辨率要求 ───┼─▶  巡航任务覆盖范围与优化系统  ──▶ 最优方案(高度/航线/拍照/电量/存储/传输)
气象数据 ─────┤    (覆盖模型 + 多目标优化 +      ──▶ 可行性 + 推荐方案
飞机档案(课题1)┘   实时电量监控与返航决策)
                                              │
飞机实时电量 ──────────────────────────────────┴─▶ 继续任务 / 减速省电 / 立即返航
```

### 2.2 角色与使用场景

| 角色                       | 场景                                                         |
| -------------------------- | ------------------------------------------------------------ |
| 任务规划员                 | 输入区域与分辨率要求，获得「飞多高、拍多少、占多少存储、够不够电」的方案与多个候选 |
| 飞行监控模块（外部消费方） | 按系统输出的实时返航建议执行返航                             |
| 系统（自动）               | 飞行中持续比对剩余电量与返航所需电量，输出返航决策           |

### 2.3 运行环境与约束

- 部署形态：独立可部署服务 / 可嵌入库
- 机型/相机：支持多旋翼与固定翼；相机参数可配置
- 约束 C-1：**安全优先**——电量永远不得在未满足返航预留时继续任务
- 约束 C-2：规划预估 ≤ 500ms；实时电量监控决策 ≤ 200ms
- 约束 C-3：分辨率要求为硬约束，优化不得突破最小 GSD
- 约束 C-4：存储超限时必须明确告警并给出对策（降分辨率/分批/启用传输），不得静默溢出

### 2.4 核心设计思想

**关键洞察**：在满足分辨率要求的前提下，「最省电、最短航程、最广覆盖」在航高选择上是**同向的**——都倾向于在分辨率允许的范围内**尽量飞高**（单张覆盖大 → 照片少 → 航程短 → 省电 → 同等电量覆盖更广）。真正的权衡发生在：

- **分辨率 vs 任务效率**：分辨率要求卡住最高航高；
- **覆盖面积 vs 电量**：区域越大越耗电，必须和返航预留平衡；
- **拍照数量 vs 存储**：照片多→存储压力→是否实时传输；
- **实时传输 vs 电量/时间**：传输本身耗电耗时，需权衡。

因此系统的核心是一个**在分辨率/电量/存储/传输多约束下的航高与任务规模优化器**，叠加实时电量监控保证安全。

## 3. 功能需求

> 优先级定义：**P0**=必须有；**P1**=重要，首发应含；**P2**=增强，可迭代。

### FR-1 相机/传感器参数建模

| 项     | 内容                                                    |
| ------ | ------------------------------------------------------- |
| 编号   | FR-1                                                    |
| 优先级 | P0                                                      |
| 描述   | 为每种挂载相机建立参数模型，作为覆盖与 GSD 计算的基础。 |

**参数**

传感器宽高（像素与物理尺寸）、单像素尺寸、焦距、支持的照片格式（RAW/JPEG）及各自单张平均大小、最快连拍间隔、是否支持定距触发、云台俯角（默认垂直向下）。

**业务规则**

- 相机参数可复用于不同飞机；
- 单张照片大小按格式分别给出（RAW 通常数倍于 JPEG）。

**验收标准（AC）**

- AC1：可录入并查询相机参数；
- AC2：缺焦距/传感器尺寸等关键字段时无法完成覆盖计算并告警。

### FR-2 覆盖范围与 GSD 计算

| 项     | 内容                                                         |
| ------ | ------------------------------------------------------------ |
| 编号   | FR-2                                                         |
| 优先级 | P0                                                           |
| 描述   | 给定航高与相机参数，计算 GSD、单张地面覆盖、单条航带覆盖宽度。 |

**核心公式**

- `GSD = (单像素尺寸 × 航高) / 焦距`（地面单位/像素）；
- 单张地面覆盖宽 = `GSD × 图像宽度像素`；
- 单张地面覆盖面积 = `GSD² × 宽像素 × 高像素`；
- 航带有效覆盖宽度 = `单张覆盖宽 × (1 − 旁向重叠率)`。

**业务规则**

- 航高受分辨率上限约束：`GSD ≤ 任务要求最大GSD` → 推出最高航高 `H_max`；
- 云台非垂直时按投影修正覆盖。

**验收标准（AC）**

- AC1：航高 ↑ 时 GSD 同比例 ↑（分辨率 ↓），单张覆盖 ↑；
- AC2：给定 GSD 上限能正确反推最高航高。

### FR-3 航高优化（最省电 / 最短 / 最广）

| 项     | 内容                                                         |
| ------ | ------------------------------------------------------------ |
| 编号   | FR-3                                                         |
| 优先级 | P0                                                           |
| 描述   | 在分辨率约束下，确定最优航高与对应的最省电、最短航程、最广覆盖方案。 |

**优化逻辑**

- 在分辨率允许范围内（`H ≤ H_max`），航高越高越省电/航程越短/覆盖越广，三者同向；
- 因此**默认最优航高 = H_max**（即分辨率允许的最高高度）；
- 当 H_max 过高导致其他问题（如云层、空域限高、相机对焦上限、法规限高）时，取各项约束的最小值；
- 输出：推荐航高、该高度下的 GSD、单张覆盖、航带数、总航程、照片数、预计电量。

**业务规则**

- 多个高度上限取交集（分辨率 / 法规 / 空域 / 相机）；
- 输出多个候选航高（如 H_max、H_max×0.8）供权衡。

**验收标准（AC）**

- AC1：在分辨率约束内，推荐航高下的航程 ≤ 较低航高的航程；
- AC2：当分辨率放宽（允许更大 GSD）时，推荐航高升高、照片数减少、航程缩短；
- AC3：任一外部高度上限生效时，推荐航高不超过该上限。

### FR-4 航线与重叠率规划

| 项     | 内容                                                  |
| ------ | ----------------------------------------------------- |
| 编号   | FR-4                                                  |
| 优先级 | P0                                                    |
| 描述   | 在选定航高下，按覆盖区域 AOI 与重叠率生成航带与航点。 |

**计算**

- 航带数 = `ceil((AOI 宽度 / 航带有效覆盖宽度))`；
- 每条航带照片数 = `ceil(航带长度 / (单张覆盖长 × (1 − 航向重叠率))) + 1`；
- 总照片数 = Σ 各航带照片数；
- 总航程 = Σ 航带长度 + 航带间转弯；
- 默认重叠率：航向 75%、旁向 65%（可配置，三维建模任务通常更高）。

**业务规则**

- AOI 不规则时按其外接矩形或栅格化处理；
- 重叠率 ↑ → 照片数 ↑ → 存储 ↑、航程略增、耗时 ↑。

**验收标准（AC）**

- AC1：生成的航带能完整覆盖 AOI，无遗漏区域；
- AC2：重叠率调整时，照片数与航程按预期变化；
- AC3：总航程含航带间转弯，非仅直线之和。

### FR-5 拍照数量与拍摄节奏

| 项     | 内容                                                   |
| ------ | ------------------------------------------------------ |
| 编号   | FR-5                                                   |
| 优先级 | P0                                                     |
| 描述   | 计算拍照数量、拍摄间距、拍摄间隔，并校验相机连拍能力。 |

**计算**

- 拍摄间距（航向）= `单张覆盖长 × (1 − 航向重叠率)`；
- 拍摄间隔 = `拍摄间距 / 巡航地速`；
- 拍照数量 = FR-4 的总照片数；
- 任务巡航时长 = `总航程 / 巡航地速`。

**业务规则**

- 拍摄间隔不得小于相机最快连拍间隔，否则需降速或降重叠率并告警；
- 盘旋类子任务（如对特定目标环绕拍摄）单独计算其时长与拍照数，并入总任务。

**验收标准（AC）**

- AC1：拍摄间隔满足相机连拍上限，否则给出降速建议；
- AC2：盘旋时长、巡航时长、拍照数均可单独列示并汇总。

### FR-6 电量预算（调用课题 1）

| 项     | 内容                                                         |
| ------ | ------------------------------------------------------------ |
| 编号   | FR-6                                                         |
| 优先级 | P0                                                           |
| 描述   | 基于生成的航线、航高、巡航时长与盘旋时长，调用课题 1 引擎计算任务电量、返航电量预留、动态余量与可行性。 |

**计算**

- 任务电量 E_task = 爬升 + 巡航（按总航程）+ 盘旋 + 降落（课题 1 模型，含气象修正）；
- 返航电量 E_return、不确定性余量 E_margin（课题 1）；
- 可用电量 E_usable = `电池Wh × DoD × SOH`；
- 可行性判据：`E_task + E_return + E_margin ≤ E_usable`。

**业务规则**

- 区域过大导致不可行时，提示分批作业、降重叠率、提高航高、提高起飞电量等对策；
- 返航预留为硬约束，不得削减以满足覆盖。

**验收标准（AC）**

- AC1：电量预算三项（任务/返航/余量）分别列示，与课题 1 输出一致；
- AC2：区域过大时正确判定不可行并给出可操作对策；
- AC3：降重叠率或提高航高后，电量预算相应下降。

### FR-7 存储容量预算

| 项     | 内容                                               |
| ------ | -------------------------------------------------- |
| 编号   | FR-7                                               |
| 优先级 | P0                                                 |
| 描述   | 计算任务所需存储，对比设备可用存储，给出超限对策。 |

**计算**

- 总存储 = `照片数 × 单张平均大小`（按选定格式，RAW/JPEG 分别或混合）；
- 对比机载存储可用容量；
- 利用率 = 总存储 / 可用容量。

**业务规则**

- 存储超限为硬告警，对策：改用 JPEG、降分辨率、降重叠率、分批、启用实时传输清空空间；
- 区分"拍摄存储"与"传输后释放"两种模式下的可用空间动态变化。

**验收标准（AC）**

- AC1：RAW 与 JPEG 模式下存储预算差异显著且正确；
- AC2：超限时明确告警并给出至少一种可行对策；
- AC3：启用实时传输后，本地存储占用按"边传边删"模型下降。

### FR-8 无线传输决策与代价

| 项     | 内容                                                         |
| ------ | ------------------------------------------------------------ |
| 编号   | FR-8                                                         |
| 优先级 | P1                                                           |
| 描述   | 评估是否启用实时无线传输照片，计算其带宽、时间与额外电量代价。 |

**计算**

- 传输总数据量 = `照片数 × 单张大小 × 传输比例`（可全量或抽帧）；
- 传输时间 = `总数据量 / 链路有效带宽`；
- 传输电量 = `链路功耗 × 传输时间`（并入任务电量）；
- 决策：当存储不足、或需实时回传（搜救/监控）时启用；权衡传输电量对续航的影响。

**业务规则**

- 传输电量进入可行性判据，可能导致任务从可行变不可行；
- 带宽随距离/高度变化，按保守值估算；
- 支持策略：仅传缩略图、关键帧、或全部；边传边删以释放存储。

**验收标准（AC）**

- AC1：启用传输后，任务电量上升、可行性可能翻转，系统正确反映；
- AC2：带宽下降时传输时间与电量上升；
- AC3：可输出"传全部/传关键帧/不传"三档对比。

### FR-9 多目标方案优化与推荐

| 项     | 内容                                                         |
| ------ | ------------------------------------------------------------ |
| 编号   | FR-9                                                         |
| 优先级 | P0                                                           |
| 描述   | 综合分辨率、电量、存储、传输多约束，输出推荐方案与若干候选，供规划员权衡。 |

**优化目标与约束**

- 目标：在满足分辨率与覆盖完整性的前提下，最小化电量（或最大化单次覆盖面积）；
- 约束：分辨率 GSD 上限、电量可行（含返航预留）、存储不溢出（或启用传输）、法规/空域高度上限；
- 决策变量：航高、重叠率、巡航速度、照片格式、是否传输。

**输出**

- 推荐方案（综合最优）+ 2~3 个候选（如「最省电」「最高分辨率」「最大单次覆盖」）；
- 每个方案列出：航高、GSD、航程、照片数、电量、存储、传输、可行性、瓶颈维度。

**业务规则**

- 无可行方案时，明确指出"无解"及最接近可行的调整方向（缩小区域/放宽分辨率/换更大电池/分批）。

**验收标准（AC）**

- AC1：多约束下能输出至少一个可行方案或明确的"无解 + 调整建议"；
- AC2：候选方案在关键指标上可读对比；
- AC3：推荐方案不违反任何硬约束。

### FR-10 实时电量监控与返航决策

| 项     | 内容                                                         |
| ------ | ------------------------------------------------------------ |
| 编号   | FR-10                                                        |
| 优先级 | P0                                                           |
| 描述   | 飞行中持续监控电量损耗，结合当前位置的返航所需电量，实时决策"继续任务 / 减速省电 / 立即返航"。 |

**输入**：实时剩余电量、当前位置、已拍进度、实时气象（风）、飞机档案（课题 1）。

**核心比较（每个监控周期）**

- `剩余电量 E_now`（实测，含损耗率估计）；
- `E_return_here`（从当前位置返航所需，调用课题 1 的返航模型 + 实时风）；
- `E_remaining_task`（剩余未拍部分的任务电量）；
- 安全裕度 `E_buffer`（应对突发，含课题 1 的动态余量）。

**决策逻辑（状态机）**

```
if E_now - E_return_here - E_buffer < 0:
    决策 = 立即返航 (硬触发, 不可覆盖)
elif E_now - E_return_here - E_buffer - E_remaining_task < 0:
    决策 = 任务无法完成, 尽快返航 (在剩余电量允许下尽量多拍关键区/降速省电)
elif 电量损耗率 异常偏高:
    决策 = 告警 + 评估是否需提前返航
else:
    决策 = 继续任务
```

**业务规则**

- 立即返航为最高优先级，一旦触发不可被覆盖；
- 损耗率异常（实测远高于预估）时主动告警，可能预示风变/故障/老化；
- 返航建议输出为「RTH 触发电量阈值（课题1）」+「实时决策」双层：阈值用于飞控兜底，实时决策用于更聪明的提前规划；
- 任务无法完成时，优先保证已拍数据安全（是否需要先传关键帧再返航）。

**验收标准（AC）**

- AC1：剩余电量逼近返航所需时，正确触发返航决策；
- AC2：逆风导致返航能耗上升时，E_return_here 增大、返航提前触发；
- AC3：损耗率异常时产生告警；
- AC4：决策响应 ≤ 200ms；
- AC5：任务无法完成时，决策为"尽快返航"而非"强行拍完"。

## 6. 业务流程

### 6.1 规划阶段

```
输入 AOI + 相机 + 分辨率要求 + 气象
    │
    ▼
按分辨率约束求 H_max ──▶ 选定航高(默认 H_max)
    │
    ├─▶ 覆盖/GSD 计算 ──▶ 航线与重叠率 ──▶ 照片数/拍摄节奏/航程/时长
    ├─▶ 电量预算(调用课题1: E_task/E_return/E_margin) ──▶ 可行性
    ├─▶ 存储预算 ──▶ 是否超限?
    └─▶ 传输决策 ──▶ 传输电量并入任务电量
    │
    ▼
多目标优化 ──▶ 推荐方案 + 候选(标注瓶颈维度: 电量/存储/分辨率)
```

### 6.2 执行与监控阶段

```
飞行中每个监控周期:
    读取 剩余电量 E_now + 位置 + 已拍进度 + 实时气象
        │
        ▼
    调用课题1: 计算当前位置返航所需 E_return_here
        │
        ▼
    决策状态机: 立即返航? / 任务无法完成(尽快返航)? / 损耗异常告警? / 继续
        │
        ▼
    输出决策 + 原因 (供监控/飞控执行)
```

## 附录 A：核心公式速查

| 项                   | 公式                                              |
| -------------------- | ------------------------------------------------- |
| GSD                  | `GSD = (单像素尺寸 × 航高) / 焦距`                |
| 最高航高(分辨率约束) | `H_max = (焦距 × 最大允许GSD) / 单像素尺寸`       |
| 单张地面覆盖宽       | `= GSD × 图像宽像素`                              |
| 航带有效覆盖宽       | `= 单张覆盖宽 × (1 − 旁向重叠率)`                 |
| 航带数               | `= ceil(AOI宽度 / 航带有效覆盖宽)`                |
| 拍摄间距(航向)       | `= 单张覆盖长 × (1 − 航向重叠率)`                 |
| 拍摄间隔             | `= 拍摄间距 / 巡航地速`                           |
| 总照片数             | `= Σ 各航带(ceil(航带长/拍摄间距)+1)`             |
| 任务电量(调用课题1)  | `E_task = 爬升+巡航+盘旋+降落`                    |
| 可行性判据           | `E_task + E_return + E_margin ≤ E_usable`         |
| 总存储               | `= 照片数 × 单张大小`                             |
| 传输时间             | `= 传输数据量 / 有效带宽`                         |
| 传输电量             | `= 链路功耗 × 传输时间`                           |
| 实时返航触发         | `E_now − E_return_here − E_buffer < 0 ⇒ 立即返航` |
